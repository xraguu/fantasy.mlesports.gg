## Follow-up round 37 (2026-07-20) — couldn't re-fill a league slot after removing a manager: draftPosition/waiverPriority collision

- **Real bug, reported live: filled a league, removed one manager, and every subsequent attempt to add a replacement failed.** Root cause: `FantasyTeam` has `@@unique([fantasyLeagueId, draftPosition])`, but both add routes (`teams/route.ts` single-add and `teams/bulk/route.ts`) assigned a new team's `draftPosition` (and `waiverPriority`) from the current team *count* (`league.fantasyTeams.length + 1`), not from which positions were actually still free. Removing a team never renumbers the teams around it — a 12-team league that loses the manager at position 7 ends up with positions `{1..6, 8..12}`, a gap at 7 but still going up to 12. The next add then computed `count + 1 = 12` — already occupied — and the `fantasyTeam.create` (or the whole batch transaction, for bulk) threw a unique-constraint violation on every single attempt, not just the first.
- Fixed both routes to track the actual set of occupied `draftPosition`/`waiverPriority` values and assign the lowest open slot instead of `count + 1` — which also means a replacement manager naturally lands in the vacated spot rather than being tacked onto the end.
- Left the DELETE route's behavior alone (doesn't renumber the remaining teams' positions after a removal) — renumbering existing teams' draft positions has its own downstream implications (draft order display, historical pick records) that weren't part of what broke here; filling gaps on add resolves the actual reported failure without touching that.

Verified: `npx tsc --noEmit` clean, full `npm run build` clean (dev server stopped first), dev server restarted and confirmed responding. Found the exact live repro already sitting in the database — a third test league ("Test v2.0.0", 7/8 teams, gap at position 1 from an earlier removal) — and confirmed mechanically that the old logic would have assigned position 8 (colliding with the existing team there, matching the reported failure exactly) while the new logic correctly assigns position 1, filling the gap.
