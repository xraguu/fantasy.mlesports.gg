  - `importMatches()`'s `Match.week` field has its own hardcoded/wrong day-math (pre-existing `// TODO`), but confirmed via grep that `Match.week` is never read anywhere in the codebase — left alone as dead code, not worth the churn.

## Follow-up round 17 (2026-07-16) — all three waiver systems actually work now

Audited the waiver-claim system first and found it was mostly scaffolding: `FantasyTeam.waiverPriority` was set once at team-join time (join order) and never touched again — no reverse-draft-order initialization, no weekly reset, no "move to back of line on a win," and FAAB bids were never actually compared between competing claims (processing was a flat priority-ordered list for every waiver system, including FAAB). The claim modal had no FAAB bid input at all. Built all of it for real:

- **`lib/waiverPriority.ts`** (new): `initializeWaiverPriorityFromDraftOrder` (reverse draft order, called at draft completion — both the normal last-pick path in `lib/draftPick.ts` and the admin "Skip Draft" route), `moveTeamToBackOfWaiverLine` (renumbers the whole league 1..N so a winner goes to the back while everyone else keeps their relative spot — a team with no successful claims never loses ground), `resetWaiverPriorityToReverseStandings` (Fixed Order only).
- **`lib/waiverProcessing.ts`** (new): the actual claim-processing algorithm, extracted from the admin route so both it and a new lazy auto-sweep share one implementation. Rolling/Fixed leagues process in live `waiverPriority` order (not the stale priority snapshotted at submission time — a real staleness bug the old code had) and rotate the winner to the back. FAAB leagues resolve differently: claims are grouped by contested MLE team, highest bid wins (ties broken by priority, then submission time), losers get denied as outbid — no priority movement, since FAAB doesn't use one.
- **Fixed Order actually resets weekly now**: `lib/autoLock.ts`'s existing week-boundary sweep (same 3am ET trigger as lineup locking) now also resets Fixed Order leagues' priority to reverse standings-through-the-just-finished-week, once per (league, week) via the same `WeekLockEvent` dedup table (new `"waiver_reset"` event type). Rolling leagues are untouched by this — they're never supposed to reset.
- **Real automatic waiver processing**: `SeasonSettings.waiverSchedule` (already existed in the schema/admin UI, but was never actually wired to anything — claims only ever processed when an admin manually clicked a button) now really drives processing. `runWaiverProcessingSweep` checks whether the current ET day/time is at or past one of a league's scheduled waiver times and processes that league's pending claims if so — wired into `GET /api/leagues/[leagueId]/waivers` and `GET /api/leagues/[leagueId]/rosters/[teamId]`, the same lazy-sweep-on-read pattern as `runAutoLockSweep`. No dedup marker needed — processing only ever touches claims still "pending," so re-checking after a day's batch is already done is a harmless no-op.
- **Manual "Run Waivers Now" button** added to the admin league detail page (shown once `draftStatus === "completed"`) as an explicit backstop for whenever the automatic sweep doesn't run for some reason.
- **FAAB bidding is now real, end to end**: `POST /api/leagues/[leagueId]/waivers` requires and validates a whole-dollar bid (can't exceed remaining budget) for FAAB leagues; the team-portal waiver claim modal has an actual bid input now (previously didn't exist at all — the backend accepted `faabBid` but nothing ever sent one).
- **Priority/budget visibility everywhere it was asked for**: the waiver claim modal shows "Your Waiver Priority: #3" or "Your FAAB Budget: $67 remaining" right in the header. My Roster's Waivers tab now has a league-wide priority order list (Rolling/Fixed) or the viewer's own remaining budget (FAAB), via a new `GET /api/leagues/[leagueId]/waiver-priority`. The My Roster header's existing priority/FAAB line (built in an earlier round) needed no changes — it was already correct, just fed by data that never actually changed until now.
- **Info modal**: new "How do the waiver systems work?" button under Waivers & Free Agents, opening a sub-modal explaining Fixed Order, Rolling, and FAAB in the exact terms specified. Also fixed a real copy bug in the Scoring sub-modal — it labeled the win/loss bonus "Series Win"/"Series Loss" when the game only scores per-game (`gameWin`/`gameLoss`) wins/losses, never a series result.
